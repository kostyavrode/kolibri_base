# FileXorProcessor

Небольшое Qt-приложение под Windows, выполняющее побитовый XOR входных файлов с 8-байтовым ключом.

### Архитектура

- **`MainWindow`**: отвечает за UI, чтение настроек с формы, запуск однократной обработки и работы по таймеру, а также за обновление статуса и прогресса.
- **`ProcessingConfig`**: простой объект-конфигурация, передаётся из GUI в рабочий поток (каталоги, маска, поведение при совпадении имён, ключ XOR и т.д.).
- **`FileProcessor`**: рабочий объект без UI, выполняет поочерёдную обработку списка файлов в отдельном потоке, читая их по кускам и применяя XOR с 8-байтовым ключом.

Используется типичный для Qt паттерн **Worker Thread**: `FileProcessor` живёт в отдельном `QThread`, общается с `MainWindow` через сигналы/слоты и не блокирует главный (GUI) поток даже при обработке больших файлов.

### Сборка

- Через qmake: открыть в Qt Creator файл `FileXorProcessor.pro`, выбрать Qt-kit с MinGW и собрать.
- Через CMake в Qt Creator: открыть корневой каталог с `CMakeLists.txt`, выбрать профиль с MinGW и убедиться, что в нём корректно настроен путь к Qt (переменная `CMAKE_PREFIX_PATH` или `Qt6_DIR`/`Qt5_DIR`).
- Через скрипт: запустить `build_mingw.bat` (должны быть в PATH CMake, MinGW и Qt, настроенные аналогично профилю Qt Creator).
